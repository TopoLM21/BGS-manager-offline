# LeadDangerous

Легкий офлайн-менеджер BGS-планов для Elite Dangerous. Приложение хранит данные
локально в браузере и синхронизирует игроков через TXT-отчеты.

## Запуск

Откройте файл `app/index.html` двойным кликом в браузере. Установщик, сервер и
зависимости не нужны.

Данные сохраняются в `localStorage` текущего браузера.

## Основной поток

1. Укажите свой позывной в шапке.
2. Начните вводить название системы.
3. Выберите нужную систему из списка EDSM.
4. Выберите фракции EDSM, для которых нужен план влияния.
5. Задайте срочность, статус и свободно опишите проблему или план.
6. Откройте карточку системы и нажмите `Добавить влияние`, если нужен ручной замер: сумма должна быть 100%.
7. Записывайте действия по системам.
8. Открывайте карточку системы, чтобы видеть график, текущие задачи и список действий.
9. При необходимости нажмите `Обновить EDSM` в карточке системы, чтобы подтянуть свежие фракции и историю влияния.
10. Обменивайтесь TXT-отчетами через `Скачать TXT` и `Загрузить TXT`.

## Синхронизация

Приложение хранит не только текущее состояние, а журнал операций:

- кто добавил или изменил систему;
- кто записал действие;
- кто внес ручной замер влияния;
- когда был создан TXT-отчет;
- когда был загружен чужой отчет;
- какие операции уже применялись.

Экспорт выгружает весь известный журнал. Поэтому схема обмена простая:

1. Игрок A выгружает TXT и отправляет игроку B.
2. Игрок B загружает TXT, добавляет свои изменения и выгружает новый TXT.
3. Игрок A загружает TXT игрока B.

Один и тот же отчет можно импортировать повторно: операции имеют уникальные ID,
поэтому дубли не будут добавлены.

Кнопка `Журнал` в шапке открывает отдельное окно с операциями синхронизации.
Кнопка `Очистить данные` полностью сбрасывает локальное состояние после двух
подтверждений.

Фракции с текущим влиянием `0%` не показываются в выборе, графике и ручном
замере: они считаются неактивными для текущей работы.

## Формат отчета

Отчет остается обычным текстовым файлом. Строки с `#` — метаданные, остальные
значимые строки — JSON-события:

```txt
# LEADDANGEROUS REPORT v1
# reportId=20260619T120000-example
# exportedAt=2026-06-19T12:00:00.000Z
# source=node-example
# author=CMDR Topo
# events=2
{"schema":"LeadDangerous.Event/v1","id":"...","type":"system.upsert","payload":{"name":"Sol","problem":"Проверить влияние после тика"}}
{"schema":"LeadDangerous.Event/v1","id":"...","type":"influence.snapshot","payload":{"system":"Sol","factions":[{"name":"Mother Gaia","influence":64.2,"locked":true},{"name":"Sol Workers' Party","influence":35.8,"locked":false}]}}
{"schema":"LeadDangerous.Event/v1","id":"...","type":"activity.add","payload":{"system":"Sol","text":"Сделал 4 миссии"}}
```

## API

Поиск систем использует EDSM API `GET https://www.edsm.net/api-v1/systems`.
Список фракций и история влияния берутся из
`GET https://www.edsm.net/api-system-v1/factions?showHistory=1`.
Приложение берет из ответа название, координаты, текущую фракцию, permit,
фракции, состояние и историю влияния, если EDSM их возвращает.

## Сервисная панель

Для отладки локальной логики дважды кликните по заголовку `LeadDangerous` и
подтвердите открытие два раза. Панель показывает количество систем, событий,
замеров влияния и контрольные суммы по фракциям.
